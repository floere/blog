Well structured code lets you minimize Information flow (because if it is where it needs to be, it doesn’t need to flow), and by accident and minimizing frictional loss also makes it fast.

So, always structure your code well, as it will help you remove redundancies and minimize coupling. Also it will enhance understanding not just by you, but others as well.

You will walk away from your code daily, feeling in control and happy.

P.S: With tests you can forget large parts of the application and still feel in control (in an positive sense).

Dipping your toe in Ruby C

This is a little follow up to my presentation at EuRuKo 2010. In it, I urged people to look under Ruby’s hood, to check out the Ruby C code.

So if you have done that, and would like to get going coding your first extension yourself, I’ve put together some files that might be helpful.

The C code is nothing to look at (It just p-s what is given to it), but check out the Ruby C code of Ruby itself, for example in http://ruby-doc.org/core/classes/Array.html by clicking on the method signatures. Or looking at Ruby source code directly.

Have fun!
